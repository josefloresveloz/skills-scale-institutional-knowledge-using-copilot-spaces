# QA Checklist

**Owner:** QA Analyst  
**Purpose:** Confirm that quality gates are met before a release proceeds.

---

## Test Readiness

- [ ] Test plan created and reviewed against acceptance criteria
- [ ] All acceptance criteria are covered by at least one test case (automated or manual)
- [ ] Test environment is configured and stable

## Test Execution

- [ ] Automated test suite runs green (no unresolved failures)
- [ ] Regression tests pass for all impacted areas
- [ ] Critical user flows smoke-tested end-to-end
- [ ] New features validated against acceptance criteria

## Defect Management

- [ ] All open defects triaged (severity, priority, and owner assigned)
- [ ] No critical or high-severity bugs unresolved at release time
- [ ] Bug fixes verified by QA before sign-off

## Sign-Off

- [ ] QA sign-off documented and communicated to the Release Manager
- [ ] Any known issues or deferred defects listed in release notes

---

> **Note:** This checklist is a lightweight starting point. Adapt it to your project's testing framework and risk tolerance.
